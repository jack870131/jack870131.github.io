Approach

The top K elements pattern is a technique aims to return a given number of the most frequent/largest/smallest elements in a given set.

The key data structure of solving top K elements problems is Heap. As the feature of max heap and min heap, we can always get the largest and smallest elements efficiently, i.e., $O(logn)$.

Let’s see some examples. We will first look at a LeetCode medium problem — Kth Largest Element in an Array. It requires us to find a given number of the largest element in a set of integers.

Here are some examples:

Input: [3,2,1,5,6,4] and k = 2
Output: 5
Input: [3,2,3,1,2,4,5,5,6] and k = 4
Output: 4

Without using heap, a brute-force solution is to simply sort the array and find out the element at the Kth place. However, this method will result in $O(nlogn)$ due to the sorting function of the algorithm.

So can we improve the time complexity of this algorithm?

Definitely. Instead of sorting the array, we can maintain a K size min heap, which enables storing and sorting the visited elements at the same time.

After deciding the data structure, we can now enter the solving process. The idea of solving this problem by the top K elements pattern is to keep the size of our min heap.

Since the root element will always be the smallest within a min heap, which means that if the size of a heap is larger than K, the root element should be discarded as it is the (K+1)th largest element so far.

As a result, the algorithm process will be like:

  1. Traverse the array elements and put them into a K size min heap. If the size of the heap is larger than K, pop out the root element. Repeat this process until the end of the array.
  2. Return the root element of the min heap as it is the Kth largest element of the whole array.

A visualization of the algorithm is provided below:

Now we can easily write the code implementation:

class Solution {
  public int findKthLargest(int[] nums, int k) {
    // Create a min heap
    PriorityQueue<Integer> minHeap = new PriorityQueue<>((a, b) -> a - b);

    // Insert the array elements into the min heap.
    for (int i = 0; i < nums.length; i++) {
      minHeap.add(nums[i]);
      // Pop out the root element if the min heap size is larger than k
      if (minHeap.size() > k) {
        minHeap.poll();
      }
    }

    return minHeap.peek();
  }
}

By applying the top K elements pattern, we can improve the time complexity of the solution to $O(NlogK)$, which the extraction of the root element of a min heap is $O(logK)$, and we need to traverse the whole array.

Next, we’re gonna look at an advanced problem — Top K Frequent Elements, which the solution will involve more processes. This problem requires us to return the K most frequent elements within a set of integers. Note that the time complexity of our solution cannot exceed $O(nlogn)$ and the result elements can be in any order.

Here are a few examples:

Input: nums = [1,1,1,2,2,3], k = 2
Output: [1,2]
Input: nums = [1], k = 1
Output: [1]
